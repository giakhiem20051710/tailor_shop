Cẩm Nang Trắc Nghiệm: Core Java

Phần 1: Ma trận String & String Pool (Câu hỏi rớt nhiều nhất)

Câu 1.1: Cho đoạn code sau, kết quả in ra màn hình là gì?
String s1 = "FPT"; String s2 = "FPT"; String s3 = new String("FPT"); System.out.println(s1 == s2); System.out.println(s1 == s3); System.out.println(s1.equals(s3));
  • A) true, true, true
  • B) false, false, true
  • C) true, false, true (ĐÁP ÁN ĐÚNG)
  • D) true, false, false
Giải thích cốt lõi:
- Ký hiệu == trong Java KHÔNG dùng để so sánh chữ viết, mà dùng để so sánh ĐỊA CHỈ BỘ NHỚ (O nhớ).
- Khi báo String s1 = "FPT", chữ "FPT" được cất vào một kho báu đặc biệt gọi là String Pool. Khi tạo s2="FPT", Java qua kho báu tìm thấy chữ đó rồi, nên nó bắt s2 chỉ chung 1 ô nhớ với s1. Suy ra s1 == s2 là True.
- Khi dùng từ khóa new, Java BẮT BUỘC tạo một ô nhớ hoàn toàn MỚI nằm ngoài kho báu (trong vùng Heap). Nên s1 và s3 khác ô nhớ. Suy ra s1 == s3 là False.
- Lệnh .equals() KHÔNG quan tâm ô nhớ, nó lấy từng chữ cái ra so sánh ruột. Về mặt cái ruột (Giá trị thực) thì "FPT" giống hệt "FPT". Suy ra là True.
Câu 1.2: Tại sao String trong Java được gọi là "Immutable" (Không thể thay đổi)? Hãy xem đoạn code sau:
String s = "Hello"; s.concat(" Java"); System.out.println(s);
  • A) Sẽ in ra: "Hello" (ĐÁP ÁN ĐÚNG)
  • B) Sẽ in ra: "Hello Java"
  • C) Lỗi biên dịch (Compilation Error)
Giải thích:
Immutable nghĩa là Một khi đã sinh ra thì mãi mãi không bao giờ thay đổi bản chất. Lệnh s.concat(" Java") sẽ tạo ra một chữ "Hello Java" MỚI TINH và ném nó vào khoảng không (vì không có ai gán lại biến để hứng nó). Bản thân biến s cũ vẫn đang trỏ về chữ "Hello" nguyên thủy. (Trừ khi bạn viết: s = s.concat(" Java"); thì nó mới đổi hướng).

Phần 2: Hố đen Interface và Abstract Class

Câu 2.1: Điểm khác biệt CỐT LÕI NHẤT giữa Interface và Abstract Class trong Java (Java 8+) là gì?
  • A) Interface không thể có hàm có thân (body), Abstract Class thì được. (SAI vì Java 8 Interface đã có 'default' method)
  • B) Một class có thể implements (kế thừa) NHIỀU Interface cùng lúc, nhưng CHỈ ĐƯỢC extends (kế thừa) MỘT Abstract Class duy nhất. (ĐÁP ÁN ĐÚNG)
  • C) Biến trong Abstract Class mặc định là public static final.
  • D) Bạn có thể dùng từ khóa 'new' để tạo object từ Interface.
Giải thích kiểu đời thực:
- Abstract Class (Người Bố): Mỗi người chỉ có tối đa 1 người bố ruột. Java cấm Đa kế thừa class để tránh tình trạng "Đẽo cày giữa đường" (Diamond Problem). Nếu 2 ông bố cùng rèn cách 'đánh kiếm', thằng con không biết nên nghe theo ông nào.
- Interface (Kỹ năng nghề nghiệp / Chứng chỉ): Một người có thể học nhiều kỹ năng ngoài xã hội. Bạn vừa có thể implements ChayXeMay, BietBoi, HatHay cùng một lúc. Biến trong interface luôn bị ép cứng là public static final (Hằng số bất di bất dịch).

Phần 3: Gia Phả Bộ Sưu Tập (Collections Framework)

Câu 3.1: Nếu bạn cần một cấu trúc dữ liệu LƯU TRỮ NHIỀU PHẦN TỬ, nhưng BẮT BUỘC KHÔNG ĐƯỢC CÓ PHẦN TỬ TRÙNG LẶP. Bạn xài cái nào?
  • A) ArrayList
  • B) LinkedList
  • C) HashSet (ĐÁP ÁN ĐÚNG)
  • D) HashMap
Học thuộc lòng bảng vàng này:
📌 LIST (ArrayList / LinkedList): Là cái băng chuyền. Nhét vào sao thì in ra theo thứ tự y chang vậy. Mười số 1 nhét vô cũng nhận lấy hết (Cho phép TRÙNG LẶP).
📌 SET (HashSet / TreeSet): Giống rổ rá sàng cát. TUYỆT ĐỐI KHÔNG CHỨA ĐỒ TRÙNG LẶP (No duplicates). Bạn nhét ba số 1 vào, Set chỉ lưu đúng 1 lần số 1. (HashSet không bảo đảm thứ tự in ra).
📌 MAP (HashMap): Cặp đôi hoàn cảnh (Key - Value). Ví dụ: ("SinhVien1" - "Gia Khiêm"). Cải Key thì cấm trùng, còn Value trùng bao nhiêu cũng được.
Câu 3.2: Sự khác biệt lớn nhất về hiệu năng giữa ArrayList và LinkedList nằm ở đâu?
  • A) LinkedList ít tốn RAM hơn ArrayList.
  • B) Tốc độ truy xuất phần tử bất kỳ (get(i)) của ArrayList cực nhanh O(1), còn LinkedList phải mò từ đầu O(N). Ngược lại, LinkedList cắm thêm người vào giữa hàng (Insert) rất lẹ. (ĐÁP ÁN ĐÚNG)
  • C) ArrayList xài cho Multi-threading, LinkedList xài cho Single-threading.
Hình ảnh tượng hình:
- ArrayList (Cái tủ thuốc Bắc nhiều ngăn kéo): Mỗi ngăn kéo được đánh số từ 0, 1, 2... Cần lấy thuốc ở ngăn số 99? Nhắm mắt móc vô ngăn 99 trong 0.1 giây là lôi ra cái kịch (Thời gian Truy xuất ngẫu nhiên là O(1)). Nhược điểm: Tủ đầy thì phải thuê thợ mộc đóng cái tủ mới bự hơn rồi nhọc nhằn bê đồ qua.
- LinkedList (Trò chơi Nối toa tàu): Các bộ dữ liệu cầm tay nhau theo dây chuyền (Node nọ trỏ ruột Node kia). Muốn tìm toa số 99, ông soát vé phải đi qua mặt 98 toa trước đó (Rất chậm O(N)). Ưu điểm: Muốn nhét thêm 1 toa vào giữa? Chỉ cần móc sợi xích ra, nhét toa mới vô rồi móc cài lại là xong, rất lẹ!
