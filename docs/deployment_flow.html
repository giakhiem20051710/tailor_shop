Bản Đồ Chiến Thuật Deploy Lên VPS

So sánh hai phương pháp đưa ứng dụng Tailor Shop lên Production

1. Code Máy Tính
Viết xong tính năng, bạn Push "chữ" (mã nguồn thô) lên GitHub.
git push origin main
2. GitHub Lưu Trữ Chuỗi Mật Mã
GitHub như một ổ cứng giữ mã nguồn, chưa có gì có thể "chạy" được.
3. VPS Tải Mã & "Nấu" (Build)
VPS tải code nháp về, dùng CPU/RAM của chính nó để cài thư viện (như npm install) rồi tự "đóng cục" và chạy.

Khá nặng VPS
git pull
docker compose up --build -d
4. Lên Sóng qua Cloudflare
Sản phẩm chín muồi nối thẳng ra tên miền asao.vn phục vụ khách hàng.

Đánh giá Phương Án 1

  • Ưu điểm: Quá dễ để thiết lập lúc ban đầu. Viết file docker-compose.yml là VPS làm hết mọi thứ, không phải đăng ký nhiều tài khoản ngoài lề. Nếu bạn chọc thay đổi ở 1 dòng nhỏ (như link URL), việc kéo về rất là mau lẹ.
  • Nhược điểm: Rủi ro cho VPS yếu! Nếu web đang có khách truy cập mà bạn gõ lại docker build, VPS sẽ "ná thở" (CPU lên 100%) dễ gây sập trang web vài giây do kẹt băng thông. Kẻ thù của VPS RAM yếu. Tốc độ build lại mất khá lâu cho NodeJS.
1. Code & "Nấu Chín" Ở Nhà
Máy tính bạn (cấu hình siêu mạnh i7, RAM 16GB) tự đóng gói code thành "viên thuốc" Image hoàn thiện.
docker build -t tailorshop:v2 .
2. Quăng Lên Kho (Docker Hub)
Thay vì đẩy "chữ" (code) lên GitHub, bạn tải 1 "cục máy đã chạy sẵn" lên kệ để lưu (Docker Hub như một Google Drive cho ứng dụng).
docker push tailorshop:v2
3. VPS Tải & Bật Chạy Chớp Nhoáng
VPS tải cục thuốc chín về và nhai luôn mà không mệt nhọc tí xíu nào (Không tốn CPU giải nén thư viện nhọc nhằn). Cập nhật tính bằng giây.
docker pull tailorshop:v2
docker compose up -d
4. Lên Sóng qua Cloudflare
Ứng dụng vừa ra mắt trong nháy mắt mà người dùng asao.vn không bị giật lag một giây màng hình trắng nào.

BẢN NÂNG CẤP XỊN SÒ: Phương Án 2 (Chuẩn Công Ty IT)

  • Ưu điểm tuyệt đối: VPS chạy cực kỳ êm ái, nhàn rỗi. Tốc độ chuyển đổi website (Deploy mới) từ lúc ấn lệnh cực nhanh. Tính bảo mật rất cao: Không cần để lọt Source Code Nháp trên VPS cá nhân mà Hacker có thể ăn cắp được (họ chỉ ăn cắp được vỏ bọc Docker đã bị mã hóa). Chứa được nhiều phiên bản. Nếu lỗi V2, có thể bấm trả lùi lại V1 trong 1 giây đồng hồ!
  • Khuyết điểm nhỏ: Tốn nhiều bước làm lúc đầu: Tạo tài khoản Github + tạo thêm tài khoản Registry. Quản lý phiên bản rối rắm hơn 1 xíu.
  • Khi nào nên dùng?: Trong project vừa hoặc lớn, các nhóm Agile (phát triển phần mềm) dùng tính năng CI/CD kết hợp cái này: Cứ code đẩy lên Github -> Máy chủ tự động build âm thầm ở hậu phương -> Đẩy cục chín ra Dockerhub -> Bắn tín hiệu sang VPS cập nhật siêu tốc!
