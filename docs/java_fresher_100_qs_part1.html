Topic 1: Toán Tử, Khởi Tạo & Short-circuit
CÂU 1: Hãy dự đoán kết quả in ra của đoạn code sau:
int a = 2; int b = 3; int c = a++ + ++b; System.out.println(a + " " + b + " " + c);
  • A) 2 4 6
  • B) 3 4 6
  • C) 3 4 7
  • D) 2 3 5
Xem Đáp Án & Giải Thích
✅ Đáp án Đúng: B) 3 4 6

Giải thích: Hãy bóc tách dòng int c = a++ + ++b;
- a++ (Hậu tố): Dùng 'a' để tính toán TRƯỚC (tức là dùng số 2). Sau đó TĂNG NGẦM 'a' lên 3. Vậy lúc này a = 3.
- ++b (Tiền tố): Tăng 'b' lên TRƯỚC (b từ 3 -> 4). Sau đó dùng số 4 để tính toán. Vậy lúc này b = 4.
- Tổng kết c = (Giá trị a đem dùng) + (Giá trị b đem dùng) = 2 + 4 = 6.
Kết quả cuối cùng: a=3, b=4, c=6.
CÂU 2: Cú lừa Ngoạn Mục của Toán Tử AND &&
int x = 5; boolean result = (x > 10) && (++x < 20); System.out.println(x);
  • A) 5
  • B) 6
  • C) Compilation Error
  • D) Exception Thrown
Xem Đáp Án & Giải Thích
✅ Đáp án Đúng: A) 5

Giải thích: Đây gọi là tính năng Short-Circuit (Ngắn mạch).
Phép toán && bắt buộc CẢ HAI vế phải True. Nhưng Java thấy vế TÂY TRÁI (x > 10) là False rồi (vì 5 làm sao lớn hơn 10).
Do đó, vòng đời chết yểu ngay lập tức. Lệnh bên VẾ PHẢI (++x < 20) CHƯA BAO GIỜ ĐƯỢC CHẠY.
Suy ra, x không bị tăng lên, vẫn yên vị là số 5!
CÂU 3: Mập mờ String Concatenation (Nối chuỗi)
System.out.println("Java" + 1 + 2); System.out.println(1 + 2 + "Java");
  • A) Java3 và 3Java
  • B) Java12 và 12Java
  • C) Java12 và 3Java
  • D) Java3 và 12Java
Xem Đáp Án & Giải Thích
✅ Đáp án Đúng: C) Java12 và 3Java

Giải thích: Thứ tự thực hiện tính từ TRÁI SANG PHẢI.
- Dòng 1: Gặp chữ "Java" đầu tiên -> Chế độ Nối Chuỗi kích hoạt. "Java" + 1 = "Java1". Sau đó "Java1" + 2 = "Java12".
- Dòng 2: Gặp số 1 + 2 đầu tiên -> Tính Toán Học trước! 1+2 = 3. Sau đó mới gặp chữ "Java" -> Nối vô thành "3Java".

CÂU 4: int vs Integer (Boxing) cực kỳ dễ nhầm lẫn!
Integer num1 = 100; Integer num2 = 100; Integer num3 = 500; Integer num4 = 500; System.out.println(num1 == num2); System.out.println(num3 == num4);
  • A) true, true
  • B) true, false
  • C) false, false
  • D) false, true
Xem Đáp Án & Giải Thích
✅ Đáp án Đúng: B) true, false

CÂU NÀY 90% FRESHER GỤC NGÃ!
Giải thích: Java có bộ đệm (Integer Cache) tự động sinh ra cho các Object Integer có giá trị từ -128 đến 127.
- Số 100: Nằm trong vùng Cache. Cho nên `num1` và `num2` dùng CHUNG một ô nhớ rỗng có sẵn. Suy ra == trả về True.
- Số 500: Vượt mốc 127. Java bắt buộc tạo ra 2 Object MỚI TINH cấp trên vùng Heap (Giống từ khóa new). Suy ra num3 và num4 là 2 Object khác nhà (khác địa chỉ ô nhớ). Dấu == trả về False. (Lẽ ra phải dùng num3.equals(num4) thì mới true).

  • A) Chạy ngon, in ra "FPT University"
  • B) Lỗi ở dòng 2: Do sb là final, không được sửa đổi nội dung.

Nhưng nó KHÔNG CẤM THAY ĐỔI ĐỒ ĐẠC BÊN TRONG (Trạng thái nội tại). Nên dòng 2 sb.append() sửa lại ruột là hoàn toàn hợp lệ!

CÂU 6: Bạn nghĩ Hàm Khởi Tạo (Constructor) nào của Java sau là ĐÚNG LUẬT?
  • A) public void MyClass() { ... }
  • B) public MyClass() { ... }
  • C) public static MyClass() { ... }
  • D) final MyClass() { ... }
    • Xem Đáp Án & Giải Thích
    ✅ Đáp án Đúng: B) public MyClass() { ... }

    Giải thích: Luật thép của Constructor:
    - Tên phải giống y hệt tên Class.
    - KHÔNG CÓ KỂU TRẢ VỀ (Return type). Kể cả chữ void cũng cấm! Nếu bạn thêm `void` như câu A, Java tưởng đó là một hàm (Method) bình thường tình cờ trùng tên với Class.
    - Không được dùng static, final, abstract cho Constructor.
    CÂU 7: Tính Đa Hình Mờ Ảo (Polymorphism & Overriding)
    class Animal { void sound() { System.out.println("Grrr"); } } class Dog extends Animal { void sound() { System.out.println("GauGau"); } void fetch() { System.out.println("Choi nem dia"); } } public class Test { public static void main(String[] args) { Animal myDog = new Dog(); myDog.sound(); // Dòng 1 myDog.fetch(); // Dòng 2 } }
    • A) Chạy được hết, in ra: GauGau & Choi nem dia
    • B) Dòng 1 in ra "Grrr", Dòng 2 báo Lỗi
    • C) Dòng 1 in ra "GauGau", Dòng 2 báo Lỗi Biên Dịch (Compilation Error)
    Xem Đáp Án & Giải Thích
    ✅ Đáp án Đúng: C) Dòng 1 in ra "GauGau", Dòng 2 báo Lỗi

    Giải thích: Ép kiểu từ Lên thành Cha: Animal a = new Dog()
    - Về mặt bản chất lúc chạy (Runtime), nó là 1 con Chó thực thụ, nên a.sound() sẽ thực thi hàm sủa của con Chó (GauGau). (Tính Đa hình Overriding).
    - NHƯNG về mặt lý thuyết lúc gõ code (Compile Time), biến a đang bị dán mác mác là Animal. Lớp Animal nghèo nàn không hề có kỹ năng fetch(). Compiler Java không chịu nổi sự vô lý này nên sẽ gạch chân đỏ ném Mới Dòng 2 ngay lập tức CẤM CHẠY!
    CÂU 8: Sự kỳ diệu của khối static và constructor
    class Demo { static { System.out.print("1 "); } { System.out.print("2 "); } Demo() { System.out.print("3 "); } } public class Test { public static void main(String[] args) { new Demo(); new Demo(); } }
    • A) 1 2 3 1 2 3
    • B) 1 2 3 2 3
    • C) 2 3 1 2 3 1
    • D) 1 3 2 1 3 2
    Xem Đáp Án & Giải Thích
    ✅ Đáp án Đúng: B) 1 2 3 2 3

    Giải thích mức độ load của Java:
    1. Khối static {}: Được kích hoạt ĐẦU TIÊN KHI NẠP CLASS. Và nó CHỈ CHẠY ĐÚNG MỘT LẦN DUY NHẤT TRONG ĐỜI kể cả bạn tạo 1000 object. (Nên nó in số 1 đúng 1 lần).
    2. Khối Vô Dnah khởi tạo tĩnh {}: Chạy khi băt đầu tạo Object MỚI, VÀ chạy TRƯỚC Constructor. (Vậy ra số 2).
    3. Hàm Constructor: Trùm cuối chạy cuối cùng sinh ra Object. (Ra số 3).
    -> Khởi tạo đợt 1: In 1 2 3.
    -> Khởi tạo đợt 2: Xài lại class nạp sẵn nên số 1 bay màu. Chỉ chạy tiếp 2 3.
    Topic 4: Exception Handling (Bắt lỗi)
    CÂU 9: Trò lừa kinh điển của Try - Catch - Finally
    public static int testMethod() { try { return 1; } finally { return 2; } } // Nếu gọi hàm này thì nó ra số mấy?
    • A) 1
    • B) 2
    • C) Compile Error
    • D) Runtime Exception
    Xem Đáp Án & Giải Thích
    ✅ Đáp án Đúng: B) 2

    Giải thích Cực Phẩm:
    Luật của finally là: "TRỜI CÓ SẬP, HAY LỆNH Ở TRY CÓ BẤM RETURN RÚT GỌN.. THÌ MÀY VẪN PHẢI CHẠY FINALLY TRƯỚC KHI THỰC SỰ ĐÓNG HÀM!".
    Khi khối try định Return 1 bỏ trốn, thằng finally nhảy ra chặn đầu bảo "Khoan, chờ tao làm hết cái đã!". Do finally có dán sẵn lệnh Return 2, nên nó THAY THẾ (Override) luôn ý định lúc nãy của try. Ghi đè thành rút lui theo vé số 2!
    Topic 5: Collections Framework (Rổ Khay Giỏ)
    CÂU 10: Map nào tự động động sắp xếp (Sort) các key theo vần Alphabet (A->Z) trong Java?
    • A) HashMap
    • B) LinkedHashMap
    • C) TreeMap
    • D) HashTable
    Xem Đáp Án & Giải Thích
    ✅ Đáp án Đúng: C) TreeMap

    Phân tích Gia Đình MAP:
    - HashMap: Gom rác quăng bừa, lộn xộn không có thứ tự. Nhanh nhất (O(1)).
    - LinkedHashMap: Thằng nào nhét vào TRƯỚC thì lát in ra hiển thị TRƯỚC (Nhớ thứ tự Nhét Vào). Giống hàng xếp tiền mua vé xem phim.
    - TreeMap: Là một thằng có Bệnh Sạch Sẽ! Cứ hễ thấy Data mới chui vào là nó xách cuốc cào lại cây cấu trúc thuật toán Red-Black Tree để LUÔN LUÔN SẮP XẾP KEY TĂNG DẦN (Abc, 123). Bù lại việc lấy ra chui vào lâu hơn (O(logN)).
    CÂU 11: Có Lỗi Điên Dại nào xảy ra ở dòng 2 không?
    List<String> list = new ArrayList<>(); list.add(1, "Java"); // Ném Index Số 1 vào danh sách Rỗng???
    • A) Được, nó tự dời cái xe "Java" tới slot thứ 1, slot số 0 bỏ trống Null.
    • B) Báo Lỗi IndexOutOfBoundsException
    • C) Báo lỗi Biên Dịch (Compilation Error)
    Xem Đáp Án & Giải Thích
    ✅ Đáp án Đúng: B) Runtime Lỗi IndexOutOfBoundsException

    Giải thích: ArrayList là cấu trúc dữ liệu LẤP CHỔ LIÊN TỤC. Hiện tại list mởi đẻ ở size = 0.
    Làm sao bạn có thể cắm 1 lá cờ ở tọa độ số 1 trong khi tọa độ gốc 0 của quốc gia này chưa được gieo rắc cờ. Lệnh .add(index, value) MẶC ĐỊNH bắt buộc Index <= Size của mảng . Nghĩa là slot tiếp theo cao sang nhất cũng chỉ được bằng 0 thôi! Không có chuyện nhảy cóc!

    CÂU 12: Đâu là sự khác biệt SỐNG CÒN giữa 'WHERE' và 'HAVING' khi chơi chung với GROUP BY?
    • A) WHERE dùng cho Cột là Text, còn HAVING dùng cho cột là Số Tự Nhiên.
    • B) WHERE Lọc data TRƯỚC KHI Nấu Lẩu (Group By), còn HAVING Lọc thành phẩm SAU KHI Đã Nấu Lẩu xong (Group By tính gộp).
    • C) Hai cái này xài đổi qua lại y chang nhau.
    Xem Đáp Án & Giải Thích
    ✅ Đáp án Đúng: B

    Giải nghĩa siêu thực:
    Ví dụ bạn có cty TGDĐ ngàn nhân viên. Bạn nhóm (GROUP BY) nhân viên theo Phòng Ban, và xem Phòng nào có TỔNG LƯƠNG (SUM(luong)) > 1 Tỷ:
    - Bạn DÙNG LỖI WHERE SUM(luong) > 1 Tỷ: Sai toét! Vì WHERE chạy đầu tiên khi chưa có ông nội nào nhóm lại tính Tổng Tống thì Cột Tổng không có số!
    - Đúng chuẩn là WHERE luong_tung_nguoi > 5_000_000 (Loại tụi thực tập ra) -> GROUP BY PhongBan (Gom phòng) -> HAVING SUM(luong) > 1 TỶ (Múc tiếp loại phòng Bèo Bèo Cơm).
    CÂU 13: Cú lừa Ngoạn Mục OUTER JOIN ngây ngô:

    Bảng KHACHHANG (10 dòng), Bảng ORDERS (Đơn Hàng) (2 dòng). Một khách lác đác mua đơn. Nếu xài SELECT * FROM KHACHHANG LEFT JOIN ORDERS ON... thì kết quả ra TỐI THIỂU là mấy DÒNG?

    • A) 2 dòng
    • B) 8 dòng
    • C) 10 dòng
    • D) 12 dòng
    Xem Đáp Án & Giải Thích
    ✅ Đáp án Đúng: C) Ít nhất 10 dòng!

    Quy Tắc Vàng của LEFT JOIN:
    Thằng nằm BÊN TRÁI Hợp Đồng (Ở đây là Bảng KHACHHANG) DÙ CHO có nghèo khổ FA ế chổng gọng Không Tồn Tại cái Đơn nào (Null bên Orders), thì tao cũng MANG MẶT MÀY LÊN HIỂN THỊ PHÙ DUNG cho bằng hết.
    Do KHACHHANG có sẵn 10 mống, LEFT JOIN bảo toàn mặt mũi nguyên gia súc bên trái, nên TỐI THIỂU sẽ lấy ĐỦ MẶT 10 Ả ĐÀO (và nếu khách đó mua 3-4 đơn thì bảng nở ra tới 12 13 dòng).
