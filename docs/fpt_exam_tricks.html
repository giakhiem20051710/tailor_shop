Phần 1: Technical Code (Code Chạy Testcase)

🟢 Bài 1 (Math): Tìm Ước Chung Lớn Nhất (UCLN - GCD) của 2 số int

Bí kíp: Đừng dùng vòng lặp trừ từ từ (quá chậm). Hãy dùng Thuật toán Euclid (Sử dụng phép chia lấy dư %). Nếu a hoặc b bằng 0, UCLN là số khác 0 còn lại.

public static int findUCLN(int a, int b) { // Đảm bảo số dương (Math.abs) vì UCLN luôn dương a = Math.abs(a); b = Math.abs(b); // Thuật toán Euclid tối ưu while (b != 0) { int temp = b; b = a % b; // Lấy số lớn CHIA DƯ cho số nhỏ a = temp; // Đẩy số nhỏ lên làm gốc cho lần lặp tiếp theo } return a; }
⚠️ Nhớ kỹ: Toán tử % (Modulus) là vũ khí hạng nặng cho mọi bài toán Array/Math. Cứ nhắc tới UCLN là gõ "b = a % b".
🟢 Bài 2 (String/Array): Trả về boolean xem các ký tự của String 1 có nằm hết trong String 2 không?

Bí kíp: Bài này dịch ra tiếng Anh là "Anagram" hoặc "Subset checker". Không được dùng 2 vòng lặp for lồng nhau (O(N^2) quá chậm, máy rớt testcase).
Hãy dùng Mảng đếm tần suất (Frequency Array) 256 phần tử đại diện cho bảng mã ASCII.

public static boolean isS1InS2(String s1, String s2) { // Bước 1: Tạo cái rổ đếm ký tự của kho S2 int[] countS2 = new int[256]; // Bỏ đồ kho S2 vô rổ for (int i = 0; i < s2.length(); i++) { char c = s2.charAt(i); countS2[c]++; // Ví dụ: Chữ 'A' cất vô ngăn 'A' + 1 cái } // Bước 2: Đi siêu thị lấy rổ của S2 trả tiền (Duyệt S1) for (int i = 0; i < s1.length(); i++) { char c = s1.charAt(i); countS2[c]--; // Lấy chữ nào ra thì trừ kho chữ đó // Nếu ngăn tủ bị ÂM -> Tức là S1 đòi món đồ mà kho S2 không có! if (countS2[c] < 0) { return false; // Toang! Báo false ngay } } return true; // Duyệt êm xuôi nghĩa là đồ gì của S1 thì S2 đều có đủ mớ }
🌟 Tuyệt chiêu: Dùng mảng đếm int[] count = new int[256]; là chiêu thức kinh điển của dân FAANG (Google, FB) để giải bài toán Chuỗi O(N).

Phần 2: Cạm Bẫy Trắc Nghiệm (Core Syntax & OOP)

🔴 Bẫy 1: Sự Mập Mờ Giữa `sum(int a, int b)` và `sum(Integer a, Integer b)`

Khác nhau hoàn toàn về BẢN CHẤT LƯU TRỮ và HIỆU NĂNG!

Đặc điểm Kiểu Nguyên Thủy int Kiểu Đóng Gói (Wrapper Class) Integer
Trọng lượng Siêu nhẹ (Chiếm đúng 4 byte). Tính toán thẳng trên CPU. Siêu nặng (Là một Object/Cục gạch). Nằm trên Heap Memory.
Giá trị Mặc định Mặc định là 0. Mặc định là null. (Giúp ta dễ kiểm tra khách có nhập tuối hay bỏ trống không).
Dùng cho Collections? CẤM DÙNG! List<int> sẽ báo LỖI BIÊN DỊCH. BẮT BUỘC! Chỉ chơi với List<Integer>.
Lưu ý: Bàn tay vô hình của Java sẽ tự động biến int thành Integer (Autoboxing) và ngược lại (Unboxing) nếu bạn viết lộn xộn, tuy nhiên quá trình nhảy biến này ngốn RẤT NHIỀU HIỆU NĂNG (Gây chậm rùa nếu nằm trong for 1 tỷ vòng).
🔴 Bẫy 2: "Tiền tố" và "Hậu tố" (Prefix vs Postfix)

Họ cực kỳ thích hỏi cái Đoạn Code In Ra Output liên quan đến ++i và i++.

int a = 5; int b = ++a; // Tiền tố (Cộng trước, Dùng sau). a biến thành 6 ngay, rồi nhét 6 vô cho b. (b=6, a=6) int c = a++; // Hậu tố (Dùng trước rồi mới Cộng sau). Móc c = 6. Xong việc mới âm thầm tăng a thành 7. System.out.println(a + " " + b + " " + c); // Output: 7 6 6
🔴 Bẫy 3: Ưu Tiên Toán Tử (Bodmas & Logic)

Thứ tự ưu tiên đụng độ: NGOẶC () >> NHÂN/CHIA/DƯ (*, /, %) >> CỘNG/TRỪ (+, -) >> SO SÁNH (>, <) >> CHỮ VÀ (&&) >> CHỮ HOẶC (||).

Cú lừa ngoạn mục nhất của Java là chữ `&&` (Và Ngắn Mạch):

int x = 0; if (false && (++x > 0)) { } System.out.println(x); // BẠN ĐOÁN X LÀ MẤY??? LÀ 0, CHỨ HIÔNG PHẢI 1 ĐÂU NHA!!
Short-circuit Logic: Vì chữ && bắt buộc cả 2 bên phải đúng mới Tới. Thằng Java nhìn thấy vế trái là false rồi, nó biếng đổi, NÓ BỎ QUA LUÔN VẾ PHẢI chả thèm chạy đoạn `++x`. Nên rốt cục x vẫn mù u bằng 0! Nếu là dấu || (Hoặc), nó thấy vế trái True, nó cũng thèm đếch chạy rế phải.
